# SHEET TEMPLATE v4 — per-chapter production sheet, THE PAINTED BOOK shape

**Status: DRAFT until frozen from the as-built ch01 at M4 (doc 31 §8). Evolves the doc-28 §6
seven-section sheet for the painted-platformer shape. The PROCESS is the template; the content
never is (doc 30 §1.4). Existing chNN.md sheets are Keen-era: mine their §1 unit audits,
structure inventories, and guardian identities — do not re-derive them.**

Every chapter sheet has these sections, in order:

## §1 · Identity + palette card
Chapter/unit ids · titleEn/titleDe · the unit's SB/WB page range · guardian identity ·
**palette card** (1 dominant hue pair + 2 supports + the invariant book constants), authored
from the REAL SB pages and quoted here (the card text is pasted into every commission card of
this unit — CP-14).

## §2 · Unit audit
Vocab fields/sets · grammar structures (with corpus counts) · the "book already plays our game"
hooks found on the pages · the unit lexicon file path (`grounding/uNN-lexicon.json`) and its
state. Machine-checkable ids only (the sheet checker verifies against the corpus).

## §3 · Bewitchment concept
What the antagonist's magic did to THIS unit's content (one paragraph, kid-concrete) ·
`goalDe` + `whyDe` in register (doc 29) · what visible restoration is.

## §4 · Role casting table
| role | casting (unit item) | why it's the unit (page ref) | — every enemy/platform/hazard/
interactable/currency/cage casting, all unit-grounded. Nothing generic. 2–3 enemy roles per
phase, one pattern for the arena.

## §5 · Phase-chain layout
Per phase (p1/p2/p3): the one spatial idea · glyph-map sketch or dimensions · encounter density
· currency trails (where they lead) · the Pixel-sketch checkpoint · which of the 6 cages sits
where (cookbook v2 §3 conventions: person / 2 trail / later-verb / teased / arena) · the exit.
Plus: which verb this chapter grants or stress-tests (doc 31 §4).

## §6 · Guardian arena
The deflect-back pattern (what it throws = which unit object) · telegraph/window/knot script
values per tier · the counter-window task types · the cry + console beat copy · the ally cameo
it earns later.

## §7 · Task set + story beats
The `paint/chNN.tasks.json` plan: which existing items carry (storyDe surgery list), which are
new · the door/restore/redemption-tag beat scripts · the freed-classmate line (grounded) ·
Emil-note text if this chapter carries one.

## §8 · Asset list + gates
The chapter's commission stems by class (sprites/strips/plates/cage pair/arena backdrop/beats)
against the batch plan · the gate checklist: grounding ✓ sheet checker ✓ machine playtest ✓
blind-solve ✓ Koki act gate ✓ deploy-truth hash quoted.
