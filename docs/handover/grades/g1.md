## 3. The game: "The Lost Pages"

### Premise

`g1.st.lost-pages` (`story@1`, title *The Lost Pages / Die verlorenen Seiten*). A magic storybook is **fading** ‚Äî its pages have gone empty and the words are lost. **Finn**, a small paper-and-ink sprite who lives in the book, is your guide. Each unit is one **empty page**; you and Finn walk into the room, **find the lost words** (vocab), **fix the sentence** (grammar), and **recap what happened** (comprehension) to restore the page. Finn's arc closes in unit 15: with the book whole, he *leaves the book to meet you at school* ‚Äî handing the reader back to the real classroom. It is deliberately gentle and cosy (age 6‚Äì7), no jeopardy.

Every chapter follows the same 8-beat shape (unit 15 has 9): empty-page reveal ‚Üí cameo cries for help ‚Üí **find word A** ‚Üí **find word B** (or fix-it) ‚Üí **fix the grammar** ‚Üí page restored + **recap** ‚Üí cameo thanks you ‚Üí "next room." One `taskSlot` per scene at most; `speaker` alternates Finn and the zone cameo.

### The theming engine (`packages/art-gen/src/theme.ts`)

This is the heart of the uplift. Every zone shares **one frozen 15√ó11 skeleton** ‚Äî one `P` (player start, centre), four `E` (encounter nodes, row-major), one `F` (the Finn NPC, top-centre) ‚Äî sitting at **identical cells in every theme**. Only **decor + colour** change per zone. Why frozen: the cosmetic save stores the player's **pixel** position, so moving the skeleton would break resume/save-compat (¬ßruntime). A `ZoneTheme` supplies:

- **`palette`** ‚Äî per-slot overrides of the 13 named `PaletteSlot`s (`floorBase`, `wallBase`, `propWood`, `propWarm`, `propLeaf`, `propDark`, `propMetal`, ‚Ä¶). `school-room` overrides nothing (the default palette *is* the classroom).
- **`extraKinds`** ‚Äî which prop tiles to paint beyond the 6 base `TILE_KINDS` (`floor`, `wall`, `path`, `accent`, `accent2`, `encounter`). Props: `board`, `plant`, `barrel`, `rug`, `stage`, `note`.
- **`layout`** ‚Äî the bespoke 11√ó15 char grid: `#` wall, `.` floor, `E`/`F`/`P` skeleton, any other glyph = a prop.
- **`props`** ‚Äî glyph ‚Üí `{ tile, solid }`. `solid:true` = a collision body (wardrobe, barrel, stage); `solid:false` = walkable decor (rug, note).

`resolveZoneTheme(generator)` maps the 15 generator strings to themes, **falling back to `school`** for anything unknown. **`DOMIGO_GREEN = "#16a34a"`** is baked in as the brand accent (matches `[data-grade="1"]` `--accent`) and is passed to `paintTileset` as the **encounter-node glow** ‚Äî a dark ring keeps the green node legible on any floor colour.

**The texture-namespacing fix (important, called out in the code):** Phaser textures are **global**. The original bug: the first zone to load "won," so every zone rendered zone-1's palette/props. The fix is in `OverworldScene.create()` ‚Äî tileset keys are namespaced by the zone's last id segment: `` tex(key) = `t-${key}-${ns}` `` where `ns = zoneId.split(".").pop()`. The **player sprite stays shared** (unnamespaced `p-{facing}`). Determinism is preserved: seed-stable speckle + fixed per-zone seeds mean the same zone always paints identically.

### The encounter system (`packages/game-core/src/encounter.ts`)

The four `E` nodes are **wandering encounters** layered *on top of* the story's `taskSlot` items. `resolveEncounterTasks(enc, {due, pool})` is pure/deterministic (SSR- and offline-safe; `@domigo/db` is a **type-only** import):

- **`source.kind:"due"`** ‚Üí take the learner's **due** items soonest-first (from `getDueRefs`, resolved server-side), skipping any not in the loaded pool (stale/overlay-dropped refs), then **top up** from the deterministic in-scope order (vocab+grammar sorted by `id`) so **a zone is never a dead end** (Law 6).
- **`source.kind:"scope-random"`** ‚Üí ignore the queue, draw straight from scope order.
- **Format filter:** grammar is gated to battle-friendly formats (`DEFAULT_BATTLE_FORMATS` = multiple-choice, gap-fill, anagram, context-picker); **vocab always qualifies** (carrier gap-fill).

In-game these render as the **"A word is fading ‚Äî bring it back!"** card. So the encounters recycle whatever the student personally owes (spaced repetition), while the `taskSlot`s are the fixed authored beats.

### The DialogueOverlay (Finn NPC) ‚Äî `PhaserGame.tsx`

Walking into the `F` tile opens `DialogueOverlay`, which plays the zone's chapter scene-by-scene as a **DOM overlay above the Phaser canvas** ‚Äî there is **no second grading path**; it reuses the one task renderer (`@domigo/task-ui`'s `GrammarItemView`/`VocabItemView`). Per scene it shows: the speaker's display name (`castNames[speaker]`, accent-coloured), `textEn`, the `scaffoldDe` line, and a **"Show word help"** chip that toggles the per-word `glosses`. If the scene has a `taskSlot`, the resolved item renders **inline** and the "Next ‚Üí" button is gated until it's answered. `scene.next` may be a string (linear), `null` (close), or a `Choice[]` (branch ‚Üí buttons) ‚Äî the branch path is wired even though G1's authored story is linear. Answers post via the injected `onAttempt` with **`mode:"game:g1"`** (optimistic; the server re-grades and queues to the offline outbox). The world layer never persists or grades itself.

### The enrichment

- **Per-zone comprehension (`.ci.`)** ‚Äî `comprehension.json` (`comprehension@1`), **15 items, one per zone** (`g1uNN.ci.*.mc.001`), all `multiple-choice`, `difficulty:1`, `strict:false`. Each is the chapter's **`recap` slot**: one plain plot question (e.g. *What does the pirate lose?* ‚Üí "His gold.") with three plausible distractors drawn from the same scene, plus `gloss`, `hintDe`, `explainDe`. This is the G1 analog of G2/G3's dialogue-comprehension sibling type. (Shipped in **#76**.)
- **The "Lost Pages" hub board (`zone-board.tsx`)** ‚Äî a plain, Phaser-free presentational component (imported via a `@domigo/game-2d/board` entry so it never bloats the hub chunk). It's the book's **restoration log + cumulative reward surface** (the G1 analog of G2's Evidence Board / G3's Season Board). Each of the 15 zones shows ‚úÖ restored / ‚óã unlocked / üîí locked, a "N / 15 pages restored" counter, and a **"The book is whole again ‚Äî every page restored!"** banner at 100%. Crucially, "restored" is derived from the **authoritative attempts ledger** (every `taskSlot` item solved, tier ‚âÝ `wrong`) ‚Äî **never** the wipeable cosmetic save (Law 2). Server passes the derived progress. (Shipped in **#77**.)
- **The art-prompt pipeline (`docs/art/build-g1-prompts.mjs`)** ‚Äî the generator+gate for the image library: a verbatim **STYLE** prefix ("bright, warm children's picture-book illustration for young children (age 6‚Äì7)‚Ä¶ a gentle glow of golden-green light where a book page comes back to life"), character **visual-locks** for Finn + all 14 cameos, and entries for a Finn avatar + `cover_title` + `end_book` + the **15 zone cards**. `compose()` builds each self-contained prompt; the gate asserts STYLE present, unique filenames, `< 5000` chars, and known characters, then writes `docs/art/g1-lost-pages-prompts.html`. **IP-free + text-free** (global no-readable-text negative). Images are generated **externally** from these prompts and dropped into `apps/web/public/art/g1/<file>.png` (resolved via `art.json`); until they land, the hub cards/cover **fall back** and the Phaser canvas stays procedural-but-themed. (Shipped in **#78**.)

### Runtime / player flow

The app mounts `<PhaserGame>` via `next/dynamic({ssr:false})` (Phaser is client-only; the package never imports `content-loader` or another game package ‚Äî content arrives as props). Boot: `PhaserGame` constructs the `OverworldScene` with the zone's seed/generator/encounter-count, wires `onEncounter`/`onNpc` to a React `overlay` state, and creates the Phaser game (`AUTO`, `pixelArt`, target 30fps, arcade physics, `Scale.FIT`). Movement is **arrow keys / WASD**; walking into a **‚ú¶** opens the encounter card, walking into Finn opens the dialogue. On answer, `clearEncounter(idx)` fades the node to 25% alpha and checkpoints; `resumePlayer()` hands control back when an overlay closes.

**Save model:** two layers, by design.
1. **Cosmetic save** (`OverworldState` = `{ zoneId, pos:[x,y], cleared:number[] }`) ‚Äî throttled to localStorage + `/api/game-save`, restored **only if it belongs to the same zone** (one save slot per grade-game). Wipeable; never authoritative.
2. **Authoritative progress** ‚Äî the attempts ledger (the outbox ‚Üí server), which drives the hub board's "restored" state and feeds the shared Leitner/review + streak systems (same `/api/attempts` path as the rest of the app).

The on-canvas caption reads: *"Arrow keys / WASD to move ¬∑ walk into a ‚ú¶ to practise ¬∑ talk to Finn."*
