## 3. The game: "The Wrong Name"

### 3.1 Premise — the fair-play moral twist

A school award medal has vanished from its glass case at Birchwood school. Student detectives **Mina & Theo** (the "Question Club") investigate. The apparent crime is a **theft** — but the fair-play twist is that it was done **to right an earlier injustice**: last year's science-medal was engraved with the **wrong name**. **Dani** was recorded as the winner, but the numbers show **Lena** actually placed first (Lena's score 9 > Dani's 7); Dani had quietly put his own name first. **Ben**, Lena's older brother, took the medal and left a new one bearing Lena's correct name — not to steal, but to surface the truth. The "culprit" is sympathetic; the real wrong is the original falsified record. The case closes with Lena's name restored, and apologies from both Ben and Dani.

The premise threads a coherent chain of physical evidence — a name-card, an open door, last year's numbers, a town map, a caretaker's key, a timeline, a second medal, a matching hand — so the deduction is genuinely fair-play (the reader can reach the conclusion from the clues). **Red herring:** Max, a loud classmate who is glad the medal is gone and lies about an alibi ("in town") that the map breaks — but he did not do it.

### 3.4 The distinctive layers

#### (a) The Clues / Evidence / Case economy — a re-skin, **no new reward pool** (Law 5)

The detective economy is **pure presentation over the existing XP/attempt system** — the "9 Laws" forbid a second currency:
- **Clues** = the item's XP, re-labelled. `TaskClue` computes `xpForTier((difficulty ?? 1) * 10, tier)` from `@domigo/engine` — the *same* formula every surface uses (correct = full, partial/close = half, wrong = 0) — and shows it as "+N clues". `hideXp` is set, so the raw "+XP" never appears.
- **Evidence Piece** = one per solved Case File (`EVIDENCE[chapterId]` in `detective-copy.ts`, e.g. ch01 → "the card", ch11 → "the same name").
- **Case Progress** = the chapter's solved-task percentage (a cosmetic progress bar).
- **Hot Trail** = a streak of consecutive non-combo-breaking answers (`breaksCombo` = close/wrong reset it). Persisted in `sessionStorage` (`domigo:g2:trail`) so it spans Case Files within one sitting; labels at 3/5/10 ("Hot trail!" → "Super detective!").

The **authoritative source is the attempt ledger**, not the save. `packages/db/src/game-progress.ts` → `getSolvedGameItemIds(db, userId, grade)` derives "solved" as *any graded attempt with `tier <> 'wrong'` in mode `game:g2`* (Law 2). `DetectiveSave` (`{chapterId, sceneId, clues[]}` via `onSave`) is explicitly cosmetic + wipeable — resetting it never loses progress, because the hub Evidence Board and the finale both re-derive from the ledger.

#### (b) German-as-hint (A1+/A2 scaffolding, Law 1)

Each scene shows the **English line** prominently; the German `scaffoldDe` and per-word `glosses` are **collapsed behind tap chips** ("Auf Deutsch?" / "Show word help") — comprehensible-input-first, German on demand. Brand words (clue/case/evidence) are glossed once in-UI (`COPY.evidenceGloss = "Evidence (= Beweis)"`). Task hints use `hintDe` (German). The design keeps English as the working language while never stranding a struggling A1+ reader.

#### (c) The persistent hub Evidence Board (`EvidenceGallery`, art.tsx)

Beyond the per-chapter case-file board (`EvidenceBoard`, a corkboard of pinned solved-clue notes), there is a **story-wide hub board**: every chapter's Evidence Piece as a pinned card, **locked (🔒 + "Case N") until that chapter is solved**. Locked cards are **spoiler-safe** — they never render the label (which would name the culprit). It is pure/hook-free so a server page can render it from `getSolvedGameItemIds`. (Landed #51.)

#### (d) Ligne-claire raster art (`art.json` + the 126-prompt library)

Art is **decoupled** via `art.json` (`story-art@1`): it maps chapters → backdrops (`set_*`), scenes → `portraits` (e.g. `mina_thinking`, `ben_guilty`) and `beats` (e.g. `beat_flashback_dani_swaps`), and taskSlots → `clues` props (`prop_card_old`, `prop_map`, `prop_key`…). The runtime resolves stems → files that exist on disk; **any missing slot falls back to procedural SVG** (deterministic FNV-1a→mulberry32, never `Math.random` — Law 9: `CharacterChip`, corkboard, `EvidenceTag`). So the game is fully playable with **zero** generated art.

The prompt library: `docs/art/build-g2-prompts.mjs` (a gated generator — asserts every prompt < 5000 chars, unique files, verbatim STYLE + character specs) → `docs/art/g2-wrong-name-prompts.html`. `docs/art/g2-art-files.json` records **`count: 126`** stems (clean-line / ligne-claire, IP-free, deliberately **text-free** — facts shown visually). 8 full-body character references are committed under `docs/art/Domigo Grade 2 story mode generated images/`. (Art wired + expanded #44/#45.)

#### (e) The complete Part-6 pedagogy upgrade (the heart of G2)

This is what makes G2 the deepest pedagogy layer. Each element and where it lives:

1. **Input → Guided → Output task ladder + variety (Phase 1a, #49).** Chapters progress from recognition (multiple-choice, matching) toward production (gap-fill, sentence-building, question-formation), and mix formats within a chapter (note the `group-sort` "sort the suspects" and `sentence-building` "say whose" slots). The tasks *are* the investigation steps.

2. **Scene-embedded variant carriers on all 15 chapters (Phases 2, #54/#55/#60).** `variants.json` re-frames one grammar item per chapter into the story's own words (e.g. ch04 `compare` → "9 is ___ (big) than 7." carrying the comparative item; ch11 `whose-it-is` → "___ name is on the medal?"). The scene's `taskSlot.variantKey` points at it; the renderer shows the **story-framed prompt** while grading the **same underlying item**. This is the "tasks-as-story" mechanism — one carrier per chapter, all 15 present.

3. **Dialogue-comprehension `.ci.` items (Phase 3, #56/#57).** `comprehension.json` holds **15** `comprehension@1` items (one per chapter, always the last task), multiple-choice, `difficulty 2`, with `hintDe` + `explainDe`. They test whether the student followed the *plot* ("Why was Lena first, not Dani?"), not just the grammar — checking real reading comprehension of the scene they just played.

4. **Spaced-retrieval "re-interview" (Phase 4, #58).** On a **fresh** chapter start (not a resume), `DetectiveGame` shows a due item first — framed as Mina & Theo "re-checking an old clue" (`reviewItems` prop, fed from the shared `getDueRefs` Leitner service). This weaves the game into the same spaced-review loop as `/review` (Law 6). Attempts here also feed the trail/streak.

5. **Scaffold fade at high units (Phase 5, #58).** `fadeHints = chapter.unit >= 10` → from ~chapter 10 the German `Tipp` is suppressed (`hideHint` threaded into every `TaskClue`), gradually removing the crutch as competence grows. Crucially the *content* is unchanged — only the optional help is withheld.

6. **Fair-play deduction choices (Phases 5/6, #58).** The ch11 deduction fork reuses the story's own `Choice[]` mechanism (not a new system) to let the player *name the culprit* — right and wrong picks both continue the story (wrong picks are gently corrected and reconverge), so it never blocks. (The ch01 narrative fork is the same primitive used for a lower-stakes "how to proceed" choice.)

7. **The "Solve the Case" finale (Phases 6/7, #59).** Completing the **last** chapter (when `finalePieces` is supplied) recaps every collected Evidence Piece — the pieces are **derived from the ledger** via `getSolvedGameItemIds`, and the verdict text scales with completeness ("Airtight — you found every piece" vs "the case had gaps"). A capstone that rewards cumulative mastery, not a fresh score.

8. **Teacher mastery view (Phase 7, #59).** `getUnitMastery(db, grade)` rolls the whole cohort's `game:g2` attempts into per-unit numbers (attempts, distinct items solved, correct rate) for the `/admin` surface. Grade-wide (teacher session `classId` null).

9. **Read-aloud (Phase 7, #59).** A 🔊 button per scene calls `speak()` → `speechSynthesis` at `en-GB`, `rate 0.85` (A1–A2 pace). No TTS provider needed; the same fallback path as the listening `AudioClip`.

Two more supporting pieces landed in the shared renderer (`@domigo/task-ui`), used by G2 but reused everywhere:
- **Forgiving retries with escalating help (Phase 1d, #52).** `useRetryGrading` (`MAX_WRONG = 3`): a wrong answer re-enables the input (nothing lost); the German Tipp surfaces on the 2nd try; the answer is revealed on the 3rd. **Only the terminal outcome** (a pass, or the final wrong) is reported via `onResult`, so each host still sees exactly one "task concluded" per item. `RetryNudge` gives the intermediate "Not quite — try again."
- **Accessibility pass (Phase 0a, #47) + pause-on-blur veil / outbox flush (Phase 0, #53):** `role`/`aria-live` on feedback, radiogroup semantics, focus management; attempts queue offline and flush.

### 3.5 Runtime / player-flow (`DetectiveGame.tsx`)

The app injects `onAttempt` (writing `mode:"game:g2"` rows through `POST /api/attempts` → `@domigo/engine` → `recordAttempt`) and `onSave`; **this layer never grades**. Per chapter the flow is:

1. **(fresh start only)** If `reviewItems` are due → a **re-interview** beat (one due clue, framed by Mina & Theo) before the case (§3.4e-4).
2. Walk `chapter.scenes` by `id` via the `next` pointer. Each scene renders: optional beat/backdrop image, a speaker panel (portrait or procedural chip, name tinted by `characterPalette`, 🔊 read-aloud), the English line, and the collapsible `scaffoldDe` / `glosses` chips.
3. If the scene has a **taskSlot** and its item is resolved and not yet done → render `<TaskClue>` (the shared `GrammarItemView`/`VocabItemView` with `hideXp`, `hideHint` per the fade rule). On the terminal result: award clues, pin the clue to the case file (`addClue`), update the Hot Trail (`onScored`), record the attempt. Then a "Continue →".
4. Else if `scene.next` is a **`Choice[]`** → render the branch buttons (secondary style); the pick sets the next scene (§3.4e-6).
5. Else → a "Next →" (or "Finish chapter →" when `next === null`).
6. `next === null` → **completion**: normally "Case File complete" (+ the chapter's Evidence Piece); on the **last** chapter with `finalePieces` → the **"Solve the Case"** recap with the verdict + `EvidenceGallery` (§3.4e-7).

`initialSave` resumes to the saved scene + clues (if it matches the current chapter). Progress bar = `clues.length / totalTasks`. All navigation is DOM state; no canvas, no server round-trip except the fire-and-forget attempt POST.
